## 数据流

### 游戏启动流程

```
用户点击模式按钮
  → launchXxxMode()（index.html）
  → 构造 URL 参数（mode/type/target/bpm/mutation/preview）
  → 跳转 game.html?mode=xxx&...
  → 解析 URL_PARAMS 初始化 TetrisEngine
  → 创建对应 GameMode 实例
  → engine.start() → spawnPiece → requestAnimationFrame 循环
```

### 设置同步流程

```
用户修改按键/DAS/ARR/预览数量
  → saveAllSettings()（index.html）
  → POST /api/save_settings（Flask）
  → database.py 更新 users.key_settings / das_settings
  → 本地 localStorage 同步
```

### 变异模式数据流

```
用户勾选变异效果
  → launchMutationMode() 收集所有 :checked 值
  → URL: ?mode=mutation&mutation=no_gravity,double_size
  → game.html 创建 MutationMode(engine, {type: mutation})
  → engine.mutations = Set(['no_gravity', 'double_size'])
  → 若含 double_size → engine.enableDoubleSize()
  → game loop 中 mutation 钩子触发对应逻辑
```

### 同IP小号检测流程

```
用户登录 → POST /api/login
  → app.py 提取 request.remote_addr 传入 db.login()
  → db.login 验证密码后：
      1. log_login() 写入 login_logs（审计用）
      2. check_same_ip() 查询同 IP 其他账号
      3. 若发现未确认的关联 → 返回 data.ip_warning = 其他用户名
  → 前端 doLogin 检查 ip_warning：
      - 确认 → POST /api/confirm_ip_link → db.confirm_ip_link() 写入 ip_links
      - 取消 → Auth.logout() 清除本地登录状态，不写入 ip_links
  → 管理后台 /api/admin/accounts 通过 GROUP_CONCAT 聚合每账号 login_logs
```
