### 1.2 7-Bag 随机算法

```javascript
fillBag() {
    // 7 种方块各一个 → Fisher-Yates 洗牌 → 返回随机排列
    const bag = [...PIECE_NAMES];
    for (let i = bag.length - 1; i > 0; i--) {
        const j = Math.floor(Math.random() * (i + 1));
        [bag[i], bag[j]] = [bag[j], bag[i]];
    }
    return bag;
}
```

每次取完一整袋后重新填充。保证 7 块内每种方块至少出现一次，避免连续出现同一方块。

### 1.3 SRS 旋转系统

- **旋转矩阵**：矩阵顺时针旋转（transpose + reverse rows），循环 4 次回原位
- **踢墙表**：JLSTZ 表和 I 表各含 8 个方向转换的 5 种偏移
- **双重旋转**：支持 CW（顺时针）、CCW（逆时针）、180 三种方向
- **二倍大模式**：不走 SRS，使用基础旋转 + 8 种 1 格偏移尝试（`[[0,0], [-1,0], [1,0], [0,-1], [-1,-1], [1,-1], [-2,0], [2,0]]`）

### 1.4 DAS/ARR 移动系统

```
首次按键 → DAS 充能（dasCounter 递增，按帧计数）
  → 充能达到 das 帧 → 进入 ARR 阶段
    → 每 arr 帧移动一次 → 直到松手
```

- **Auto-repeat 保证**：`onKeyDown` 以 `if (!e.repeat)` 守卫，防止浏览器 auto-repeat 重置 DAS
- **跨方块保持**：锁定后新方块不重置 DAS 计数器
- **ARR=0 短路**：`arr === 0` 时每帧都移动

## 2. 游戏模式（modes.js）

### 2.1 设计模式：策略模式

```
GameMode (基类)
├── NormalMode      — 经典模式（无额外逻辑）
├── SpeedMode       — 竞速模式（计时器/行数倒计时）
├── RhythmMode      — 节奏模式（BPM 驱动硬降）
└── MutationMode    — 变异模式（多选组合）
```

### 2.2 GameMode 接口

| 方法 | 说明 |
|------|------|
| `constructor(engine, config?)` | 初始化，注入引擎和配置 |
| `onLineClear(cleared)` | 消行钩子（变异模式触发砖粒生成/消失） |
| `getStatus()` | 返回 `{primary, secondary}` 用于 UI 显示 |

### 2.3 MutationMode 详解

- **配置格式**：`type` 为逗号分隔字符串，如 `"no_gravity,double_size"`
- **引擎存储**：`engine.mutations = Set(['no_gravity', 'double_size'])`
- **消行不掉落**：`clearLinesNoGravity()` — 只清除行不压缩，并检查是否触发 extra_cells/random_vanish
- **随机生成砖粒**：`spawnRandomCells()` — 在已有砖粒正上方生成
- **随机消失砖粒**：`vanishRandomCells()` — Fisher-Yates 洗牌后均匀随机删除
- **二倍大**：`enableDoubleSize()` — 替换 TETROMINO_SHAPES 为 DOUBLE_SIZE_SHAPES，保留原始 shape 供 preview

## 3. 后端 API（app.py）

### 3.1 接口列表

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | /api/register | 注册 | 否 |
| POST | /api/login | 登录，返回 token | 否 |
| GET | /api/user | 获取用户信息 | 是 |
| POST | /api/update_profile | 修改名称/密码 | 是 |
| POST | /api/save_settings | 保存按键和 DAS 设置 | 是 |
| POST | /api/sync_score | 上传游戏分数 | 是 |
| GET | /api/leaderboard | 获取排行榜 | 否 |

### 3.2 认证方案

- 登录成功返回随机 token，客户端存入 localStorage
- 后续请求携带 `Authorization: Bearer <token>` 头
- 服务端查 `users` 表验证 token

## 4. 前端渲染（game.html）

### 4.1 渲染方式

- 使用 CSS Grid 渲染棋盘（10×20 可见格），每格 32×32px
- 不依赖 Canvas，纯 DOM 操作
- `renderBoard(display)` 每帧重建棋盘 DOM 节点

### 4.2 预览区域

- Hold：4×2 格，18×18px/格
- Next Queue：4×(previewCount×3) 格，每方块 2 行 + 1 行间距
- 二倍大模式：preview 使用原始 shape 渲染（避免溢出）

### 4.3 键盘处理

```javascript
onKeyDown(e):
  if (e.repeat) → 仅执行移动，不重置 DAS
  else → 首次按下，初始化 DAS 计数器
```
