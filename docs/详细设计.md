### 1.2 7-Bag 随机算法

```javascript
fillBag() {
    // 7 种方块各一个 → Fisher-Yates 洗牌 → 返回随机排列
    const bag = [...PIECE_NAMES];
    for (let i = bag.length - 1; i > 0; i--) {
        const j = Math.floor(Math.random() * (i + 1));
        [bag[i], bag[j]] = [bag[j], bag[i]];
    }
    return bag;
}
```

每次取完一整袋后重新填充。保证 7 块内每种方块至少出现一次，避免连续出现同一方块。

### 1.3 SRS 旋转系统

- **旋转矩阵**：矩阵顺时针旋转（transpose + reverse rows），循环 4 次回原位
- **踢墙表**：JLSTZ 表和 I 表各含 8 个方向转换的 5 种偏移
- **双重旋转**：支持 CW（顺时针）、CCW（逆时针）、180 三种方向
- **二倍大模式**：不走 SRS，使用基础旋转 + 8 种 1 格偏移尝试（`[[0,0], [-1,0], [1,0], [0,-1], [-1,-1], [1,-1], [-2,0], [2,0]]`）

### 1.4 DAS/ARR 移动系统

**引擎端（tetris-core.js）**：

引擎类新增独立 ARR 计数器属性：
```javascript
this.arrCounter = { left: 0, right: 0 };
```

左右方向各有独立的 `arrCounter`，避免双向移动时相互干扰。

**游戏页面端（game.html）**：

DAS/ARR 定时器逻辑重构，采用"独立计数器 + 归零"方案替代旧版取模判断：

```
旧逻辑（已废弃）：dasCounter % arr === 0 → 存在首帧偏移 bug
新逻辑：
  首次按下 → dasCounter=1（第0帧执行首次移动）
  每帧 dasCounter++，若 dasCounter >= das：
    arrCounter++                   // 独立计数器递增
    若 arrCounter >= arr → 触发移动，arrCounter=0  // 到达阈值时归零
  松手 → arrCounter 重置
```

**时序示例**（DAS=1, ARR=2）：

```
帧0  首次按键 → dasCounter=1 → 触发移动（首次立即移动）
帧1  dasCounter=2 >= das(1) → arrCounter=1，未达阈值
帧2  arrCounter=2 >= arr(2) → 触发移动，arrCounter=0
帧3  arrCounter=1，未达阈值
帧4  arrCounter=2 → 触发移动，arrCounter=0
...
```

- **Auto-repeat 保证**：`onKeyDown` 以 `if (!e.repeat)` 守卫，防止浏览器 auto-repeat 重置 DAS
- **跨方块保持**：锁定后新方块不重置 DAS/ARR 计数器
- **ARR=0 短路**：`arr === 0` 时每帧都移动

## 2. 游戏模式（modes.js）

### 2.1 设计模式：策略模式

```
GameMode (基类)
├── NormalMode      — 基础模式（经典俄罗斯方块核心逻辑，供各模式继承）
├── SpeedMode       — 竞速模式（计时器/行数倒计时）
├── RhythmMode      — 节奏模式（BPM 驱动硬降）
└── MutationMode    — 变异模式（多选组合）
```

### 2.2 GameMode 接口

| 方法 | 说明 |
|------|------|
| `constructor(engine, config?)` | 初始化，注入引擎和配置 |
| `onLineClear(cleared)` | 消行钩子（变异模式触发砖粒生成/消失） |
| `getStatus()` | 返回 `{primary, secondary}` 用于 UI 显示 |

### 2.3 MutationMode 详解

- **配置格式**：`type` 为逗号分隔字符串，如 `"no_gravity,double_size"`
- **引擎存储**：`engine.mutations = Set(['no_gravity', 'double_size'])`
- **消行不掉落**：`clearLinesNoGravity()` — 只清除行不压缩，并检查是否触发 extra_cells/random_vanish
- **随机生成砖粒**：`spawnRandomCells()` — 在已有砖粒正上方生成
- **随机消失砖粒**：`vanishRandomCells()` — Fisher-Yates 洗牌后均匀随机删除
- **二倍大**：`enableDoubleSize()` — 替换 TETROMINO_SHAPES 为 DOUBLE_SIZE_SHAPES，保留原始 shape 供 preview

## 9. 游客模式

### 9.1 数据模型

accounts 表新增 `is_guest` 字段（INTEGER，默认 0）：
- `is_guest = 0`：注册用户
- `is_guest = 1`：游客账号（密码为空字符串，role 为 user）

### 9.2 Database 方法

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `guest_login` | guest_name, ip | dict | 创建游客账号（is_guest=1），记录登录 IP |
| `is_guest` | username | bool | 检查账号是否为游客 |

### 9.3 后端接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/guest` | 接收 `{username}`，调用 `db.guest_login()` 创建游客账号，返回用户信息（含 `is_guest: true`） |

### 9.4 安全策略

- **save_record 拒绝游客**：`/api/records` POST 时先 `db.is_guest(username)`，游客直接返回 `{success: true, message: "游客模式不保存记录"}`
- **排行榜排除游客**：`get_speed_leaderboard` SQL 增加 `AND (a.is_guest IS NULL OR a.is_guest = 0)`
- **IP 检测豁免**：`check_same_ip` SQL 增加 `AND a.is_guest = 0`，游客不与注册用户比对
- **管理员保护**：admin.html 中游客账号禁用重置密码/封禁/清除记录按钮

### 9.5 前端适配

- **默认页**：`showPage('home')` → `showPage('account')`，优先引导登录
- **游客按钮**：登录表单新增「游客模式」按钮，调用 `doGuestLogin()`
- **用户名生成**：`guest_<timestamp>_<random>` 格式确保唯一性
- **updateAccountPage**：游客隐藏修改名称/密码区域，显示「游客: xxx」
- **fetchLeaderboard**：游客显示提示文字「登录后可查看排行榜」
- **loadSettingsPage**：游客显示提示文字「游客模式暂不支持自定义设置」
- **saveRecord / saveSettings**：前端额外拦截，游客直接返回 skip

## 7. 登录IP追踪与同IP检测（原章节，保持不变）

### 7.1 数据模型

```sql
-- 登录日志：每次登录记录一条
CREATE TABLE login_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL,
    ip_address TEXT NOT NULL,
    login_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- IP 关联：用户确认小号关系后写入
CREATE TABLE ip_links (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    ip_address TEXT NOT NULL,
    username1 TEXT NOT NULL,  -- 字母序较小的账号
    username2 TEXT NOT NULL,  -- 字母序较大的账号
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 7.2 登录流程

```
POST /api/login { username, password }
  → app.py 获取 request.remote_addr
  → db.login(username, password, ip):
      1. 密码验证通过
      2. log_login(username, ip) 无条件记录
      3. check_same_ip(ip, username) 查询 login_logs
         → 找到同 IP 其他账号且 ip_links 无记录 → 返回对方用户名
      4. 结果附 ip_warning 字段
  → 前端 doLogin:
      ├─ 有 ip_warning → confirm("游戏不提倡开小号…")
      │   ├─ 确定 → POST /api/confirm_ip_link
      │   │         → db.confirm_ip_link() 写入 ip_links，继续登录
      │   └─ 取消 → Auth.logout() 清除状态
      └─ 无 ip_warning → 直接登录成功
```

### 7.3 管理后台展示

`get_all_accounts()` 通过子查询 `GROUP_CONCAT(DISTINCT l.ip_address)` 聚合登录 IP。admin.html 表格新增「登录IP」列，显示最近 3 个 IP，超过则折叠显示 `+N`。

### 7.4 关键约束

- `login_logs` 仅做审计，即使取消小号警告也会保留（不可逆）
- `ip_links` 按 username 字母序排序存储 `(u1, u2)`，已关联 IP 不再弹警告
- `check_same_ip` 仅返回**第一个**未关联的账号，不做多账号匹配
